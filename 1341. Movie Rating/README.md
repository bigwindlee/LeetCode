在这道题目中，我们涉及了一些重要的 SQL 核心知识点。以下是总结：

### 1. **聚合函数**：
   - **`COUNT()`**：用于统计每个用户评论的电影数量。
   - **`AVG()`**：用于计算某段时间内每部电影的平均评分。

### 2. **`JOIN`**：
   - **`INNER JOIN`**：我们使用 `JOIN` 来关联 `MovieRating` 表和 `Users` 表，获取用户的名字；以及关联 `MovieRating` 表和 `Movies` 表，获取电影的名称。

### 3. **分组和排序**：
   - **`GROUP BY`**：用于按用户或电影进行分组，结合聚合函数计算每个组的汇总数据（如评论数量或平均评分）。
   - **`ORDER BY`**：用于对分组后的数据进行排序。比如按照评论数量降序，字典序升序进行排序；或者按电影的平均评分降序排序。

### 4. **`LIMIT`**：
   - 用于限制结果的返回行数，例如只返回评论最多的用户或评分最高的电影。

### 5. **时间筛选**：
   - **`WHERE ... BETWEEN`**：用于筛选特定时间范围内的数据，特别是筛选 2020 年 2 月的评分记录。

### 6. **`UNION ALL`**：
   - 通过 `UNION ALL` 将两个查询结果合并成一个结果集。这在同一查询中返回多个不同数据点（比如评论最多的用户和评分最高的电影）非常有效。

### 7. **SQL 查询优化**：
   - 使用了子查询来避免冗长的重复代码，使得每个查询块聚焦于一个特定目标。
   - 通过 `LIMIT` 和排序的结合，确保在平局的情况下能正确选择字典序较小的结果。

### 总结：
这道题考察了**聚合操作**、**分组查询**、**时间筛选**、**多表联接**，以及通过 `UNION ALL` 合并不同查询结果的能力。掌握这些 SQL 技巧能够有效应对类似的复杂数据查询问题。
